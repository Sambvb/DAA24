Algorithm Concepts and Applications

1. What are the kinds of problems we see in nature?

Iteration: Compound interest, traffic lights.

Recursion: Directory size, fractals.

Backtracking: Sudoku, maze-solving.

Divide and Conquer: Merging records, sorting.

Greedy Algorithm: ATM withdrawals, activity scheduling.

Dynamic Programming: Route planning, Fibonacci sequence.

Graph Algorithms: Dijkstra (shortest path), Prim’s/Kruskal’s (MST).

Sorting Algorithms: Task and ticket sorting.

Searching Algorithms: Linear and binary search.

2. What is space and time efficiency? Why are they important?

Space Efficiency: Extra memory used by an algorithm.

Time Efficiency: Speed of task completion.

Orders of Growth:

  • O(1): Array access.
  • O(log n): Binary search.
  • O(n): Loop through a list.
  • O(n log n): Merge sort.
  • O(n²): Bubble sort.
  • O(2^n): Brute force.
  • O(n!): Generating permutations.

3. Takeaways from Chapter 2 Design Principles

  • Shortest Path Trees: Essential for navigation and optimization.
  • Partitioning: Simplifies problems for efficient solutions.
  • Balancing and Rotations: Keeps data structures efficient.
  • Edge Relaxation: Dynamically updates shortest paths.
  • Level Order Traversal: Breadth-first exploration.
  • Pruning: Reduces computational complexity.
  • Brave and Cautious Travel: Combines DFS (brave) and BFS (cautious).

4. Hierarchical data and optimization in tree structures

  • Tree: Simple hierarchical structure; inefficient if unbalanced.
  • Binary Search Tree (BST): Efficient search/retrieval, but degrades when unbalanced.
  • AVL Tree: Guarantees balance; costly rotations.
  • 2-3 Tree: Perfect balance; complex implementation.
  • Red-Black Tree: Approximate balance with fewer rotations than AVL.
  • Heap: Quick access to min/max values; inefficient for general searches.
  • Trie: Efficient string and prefix searches; high memory use.

5. Array query algorithms and their applications

  • Range Queries: Efficient sum/min/max lookups.
  • Dynamic Updates: Real-time changes using Fenwick Trees.
  • Prefix Queries: Cumulative sums.
  • Static Queries: Sparse Tables for min/max.
  • Searches: Binary search for sorted arrays.

6. Tree vs. Graph and their applications

Tree

  • Definition: Hierarchical with parent-child relationships.
  • Applications: File systems, decision trees, database indexes.

Graph

  • Definition: General node-edge structure.
  • Applications: Social networks, GPS, recommendation systems.

7. Sorting and Searching Algorithms

Sorting

  • Bubble Sort: Swaps adjacent elements until sorted (O(n²)).
  • Merge Sort: Divides and merges sublists (O(n log n)).
  • Quick Sort: Partitions around a pivot (O(n log n)).

Searching

  • Linear Search: Checks each element (O(n)).
  • Binary Search: Divides sorted data (O(log n)).

8. Importance of Graph Algorithms

  • Spanning Trees: Connect nodes efficiently (e.g., Kruskal’s, Prim’s).
  • Shortest Paths: Optimizes routes (e.g., Dijkstra, Bellman-Ford).

9. Algorithm Design Techniques

  • Pruning: Reduces unnecessary computations.
  • Lazy Propagation: Delays updates until needed.
  • Edge Relaxation: Updates shortest paths dynamically.
  • Balancing: Maintains efficiency in data structures.
  • Brave and Cautious Travel: Combines DFS and BFS strategies.
